# 算喵・算命網站範本（可販售）

這是一個**可以直接賣給命理／算命客戶**的網站範本。
目前預設品牌是「**算喵**」（LINE 算命貓），但**只要改一個地方，就能換成任何品牌**。

- 技術：純手寫 HTML + CSS + 原生 JavaScript，**沒有框架、不用安裝、不用 build**。
- **整個網站只有一個檔案**：`index.html`。直接用瀏覽器打開就能看，上傳到任何空間就能上線。
- 手機、電腦都自動排版（RWD）。

---

## 一、怎麼快速換品牌（賣給不同客戶時只改這裡）

打開 `index.html`，找到最上面這一段（大約在第 25 行）：

```html
<script id="config">
const CONFIG = {
  brand:   "算喵",              // ← 網站／品牌名字
  slogan:  "……",               // ← 首頁大標下面那句話
  masterName: "算喵",           // ← 算命師／機器人的名字
  ...
  line:    "https://line.me/R/ti/p/@yourid",  // ★ 換成客戶真正的 LINE 加好友連結
  lineId:  "@yourid",                         // ★ 換成客戶真正的 LINE ID
  ...
```

**要改的重點（打 ★ 的一定要換）：**

| 欄位 | 意思 |
|---|---|
| `brand` | 品牌名（會出現在左上角 Logo、標題、頁尾） |
| `slogan` | 首頁副標語 |
| `masterName` / `masterTitle` / `masterBio` | 命理師／機器人的名字、頭銜、自我介紹 |
| ★ `line` | **客戶的 LINE 加好友網址**（所有「加 LINE」按鈕都連這裡） |
| ★ `lineId` | 客戶的 LINE ID（顯示用） |
| `ig` / `fb` / `email` | 社群連結，**沒有就留成空字串 `""`**，該圖示會自動隱藏 |
| `gold` / `purple` | 網站主色，想換色系改這兩個色碼即可 |
| `services` | 服務項目（可自由增加或刪除一整組 `{ ... }`） |
| `process` | 「怎麼開始」四步驟 |
| `reviews` | 客戶好評（**放真人好評前，一定要先得到對方同意**） |
| `faq` | 常見問題 |
| `disclaimer` | 頁尾的免責聲明 |

改完存檔，重新整理瀏覽器就會看到新品牌。**不需要動 CONFIG 以外的程式碼。**

### 命理文章要改哪裡？
在同一個檔案往下找 `const ARTICLES = [` ，每篇是一組 `{ id, cat, emoji, title, ex, body:[...] }`，
`body` 裡每一個字串就是一段文字，想強調的字用 `<b>粗體字</b>` 包起來會變成金色。

## 二、怎麼「上線／交付給客戶」

這個網站是**一個純靜態檔案**，交付方式很自由，任選一種：

1. **最簡單**：把 `index.html` 傳給客戶，他自己上傳到他的網站空間即可。
2. **免費上線**：用 GitHub Pages 或 Netlify（把檔案拖上去）就有一個網址。
3. **自訂網域**：買一個網域（例如 `suanmiao.com`）指到上面的空間，就是正式官網。

> 小提醒：目前的免費占卜小工具（生肖／塔羅／每日籤）都是**前端隨機**、給人玩的娛樂效果；
> 真正的算命是導流到客戶的 LINE（算喵）去做，網站負責「介紹 + 建立信任 + 加 LINE」。

## 三、（你問的）怎麼把 Claude 跟 GitHub 串起來、讓它自動改網站

如果你希望「請 Claude 改網站 → 它自己 push 上去 → 網站自動更新」，做法如下：

### 方式 A：Claude Code on the web（最貼近你現在用的方式）
1. 到 **[claude.ai/code](https://claude.ai/code)** 登入。
2. 第一次會請你**授權 GitHub**：它會跳到 GitHub 的畫面，你按「Authorize / Install」，
   選擇要開放哪些 repository（例如這個算命網站的 repo）。
3. 授權完成後，在 Claude 裡選那個 repo，就能請它改檔、它會幫你 commit / push，
   GitHub Pages 會自動重新部署上線。

### 方式 B：本機用 Claude Code + GitHub（進階）
1. 電腦先裝好 **git** 並登入 GitHub（用 GitHub 官方的 `gh` 工具 `gh auth login` 最省事）。
2. 安裝 Claude Code CLI，在專案資料夾裡開它，就能直接請它改檔＋幫你 push。

### 重點：**授權是「帳號對帳號」，不是把密碼給我**
串接 GitHub 走的是 **OAuth 授權**（你在 GitHub 官網按同意），
**全程不需要、也請不要把任何密碼或金鑰貼進對話**。授權隨時可以到
GitHub → Settings → Applications 取消。

## 🔒 四、安全提醒（很重要，務必遵守）

- **不要把任何「金鑰、密碼、Token、API key、私人個資」貼給 AI 或寫進網站程式碼。**
  - 例如：GitHub Token、LINE 官方帳號的 Channel Secret / Access Token、金流密鑰、客戶名單、身分證等。
  - 這些一旦寫進網頁原始碼或公開的 repo，**任何人都看得到**，等於外洩。
- 網站上**公開的東西只放「可以給全世界看的」**：品牌名、公開的 LINE 加好友連結（`@id` 是公開的，可以放）、公開社群連結、行銷文案。
- **客戶好評要放真名前，先取得對方同意**；不確定就用「匿名客戶」。
- LINE 機器人（算喵）的**後端串接、金鑰設定**，請在 LINE 官方後台（LINE Developers）裡設定，
  **不要放進這個網站的檔案**。網站只需要那條「公開的加好友連結」就夠了。
